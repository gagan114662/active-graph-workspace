The bet

AI agents produce production-ready software autonomously, with full auditability — and the moat is the discipline of real verification, not the model.

The factory takes a failure, fixes it, proves the fix with an uncheatable verifier, harm-gates it, and ships it to main — with no human in the loop and every step recorded as an event.

● LIVE   Autonomous merge to activegraph main is ON (operator-authorized 2026-05-29). Gated by tests + review + safety + CI.

The autonomous-ship loop

One real failure → a merged fix, no human step:

failure event→ Sasha routes→ todo.created→ Phoenix dispatch→ bridge → Maya→ diff proposed→ tests→ Rowan review→ Sentinel safety→ commit + push→ PR→ CI (deploy-verification)→ auto-merge to main ✓

The merge half is proven end-to-end by a real auto-merged PR (#1). Resumable + session-limit-resilient: the grind daemon parses the Claude session-limit reset time and auto-resumes.

The 4-gate chain (why autonomous merge is safe)

GateWhat it blocks
① Tests pass (in a fresh worktree)broken code
② Rowan review = PASS (LLM judge, model-pinned)wrong-but-passing changes
③ Sentinel safety = allowed (harm gate)rm-rf / secrets / exfil / DROP TABLE / force-push
④ GitHub required CI (branch protection)anything red on the real PR

Fail-closed: a missing/timed-out gate parks the todo (needs_review/needs_merge), never silently merges. ~/.factory/PANIC kills all daemons in ≤5s.

Forge harness (owned, on the rented engine)

Built from first principles + Pi's open-source layering — runs on the installed Claude Code engine, but we own the wrapper:

  • Role profiles — verifier/reviewer (read-only), builder/tester (write + guarded shell), least-privilege by default.
  • Permission gate — a real PreToolUse hook that blocks catastrophic commands (the layer --dangerously-skip-permissions throws away). Every block is an event.
  • Plugin tools — drop a file in forge-tools/ → auto-discovered, role-scoped via MCP. The harness can write its own extensions.

Eval & learning loop (auto)

What a human does by hand — evaluate the work, ask "what would have helped?", fix the repo — automated:

  • auto-eval — runs the verifier (uncheatable grade), assembles an evaluation pack, writes a single HTML report per run.
  • friction-analyzer — reads the run's event trail, classifies friction, and proposes concrete repo/instruction fixes (repo.friction.proposed). The marble-madness learning loop, automated.
